Suppose a librarian is tasked with weeding a section of physical books. As part of the decision making process, he/she wants to consider what other libraries in the area might own that specific item. The librarian wants to peruse the stacks with a laptop and barcode reader, and get this information by scanning a book's barcode. You have been tasked with creating a web interface to make this a reality.

It's useful to break this problem down into these questions:

  • What do I have?
  • How can I get what I need with what I have?
In this application, we start with a barcode, and eventually want to end up with a list of regional libraries that own this item. There is no single Alma API that currently does this. However, we can use the Alma Scan In API to retrieve Bib/Items/Holdings data, including ISBN, from a barcode. We can then use the Worldcat Search API to get local libraries with our ISBN, using the LibraryLocations GetByISBN method.
